Slide 3 · Problem

Manual trap placement is inconsistent, hard to audit, and easy to underperform.

Operational pain

  • Blind spots in complex farm layouts.
  • High dependence on individual experience.
  • No reproducible placement standard.

Business impact

  • Extra labour from repeated re-positioning.
  • Volatile capture outcomes site-to-site.
  • Weak evidence for scaling decisions.

Slide 4 · Technical Implementation

Originality: stochastic movement + robust objective + simulation-backed optimisation.

Model pipeline

  1. Map to grid/graph representation.
  2. BFS-derived reachability and bottleneck features.
  3. Trap placement optimisation under fixed trap budget.
  4. Monte Carlo validation across movement scenarios.

Product stack

  • Python package + CLI (`biopath solve`).
  • FastAPI endpoints (`/api/solve`, `/api/benchmark`).
  • Web pitch studio with run history and evidence links.
  • Repeatable run artifacts for reviewer traceability.
